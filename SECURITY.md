# Security Policy

## Supported Versions

Security fixes are currently provided on a best-effort basis for:

- the latest commit on the default development branch;
- the most recent public release tag, once tagged releases start.

Older snapshots and unpublished experiment branches should be treated as unsupported research code.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for a suspected security problem.

Until a dedicated security inbox is set up, report vulnerabilities privately to the maintainers via
email:

- `nasonov_kv@mail.ru`
- `mikhail.kakanov@gmail.com`
- `valeriaskvo@gmail.ru`

Include the affected component, reproduction steps, and any known impact. We will aim to acknowledge
the report, confirm whether it is in scope for the repository, and coordinate a fix or mitigation
before public disclosure.
